A learning engine built around one persistent loop: work deposits into a graded concept memory that survives across sessions, and new work is reconstructed from that memory. Everything it publishes — including this page — is a projection of the structure it has learned.

What it is

An end-to-end research environment built around a memory. The engine compiles a research program's working material — notes, sources, numerics, drafts, its author's rulings — into a conceptric: a graded graph of typed concepts that persists and re-grades as the work moves. The loop runs persistently: a session reconstructs its working context from the graph, does its work, and deposits the results — findings, feedback, the author's rulings — back into the structure before it closes. Manuscripts, talks, this page and the demo are reconstructed from the graph, and the graph — never a draft — is what is stored.

What distinguishes it

The whole loop is inside, and every step of it is inspectable. Planning is data over the graph; writing is judged by separated mechanical review that refuses to mint a failing build; every paragraph traces back through plan and graph to source. And it is a theory made operational: the synthetics working drafts — statistical lawhood under finite description — supply the mathematics, the engine is that mathematics running, and early versions of the engine generated the drafts that describe it.

Why that matters

A research partner that accumulates structure instead of chat history can be audited instead of trusted. Of the falqon manuscript's 148 rendered objects, 122 trace back to the concept they argue, and all 54 equations resolve through their derivations; the 26 that do not yet resolve are listed with reasons. The aim it is built toward: a partner intelligence grown from one person's life and work, running on hardware they own.

The machine

One pipeline, five stages

Select a stage — it opens in place. Every count on this band is the falqon manuscript's, generated from the engine tree at commit .

Everything begins as ordinary files: manuscript sources, numerics with their readings files, feedback ledgers, prior drafts. No worker reads the pile flat — a dispatched agent is pointed at one node of the structure and a goal, and reconstructs the context it needs from there. The dispatch layer enforces the rule on every dispatched prompt.

Dispatched workers also run at or below the orchestrator's own model tier, and an audit walks every workflow to check it, rather than trusting the author of the dispatch.

The loop closes here: review findings, the author's rulings and reader feedback land back in the same ledgers this stage reads, so the next build starts from more structure than the last one did.

Every compiled build is read by a panel of detector classes — meta-leak, captions, apparatus, claims, house style, visuals, template, promises. Findings are typed records; the drain that consumes them validates evidence before counting anything closed, and has refused the orchestrator's own claimed closures. The loop refuses to lock while any hard gate fails.

Findings rise across these rounds because the detector suite widens as it learns the manuscript. The falqon build is still refused: the loop will not mint a release from it in this state.

This is a static page on a static host, so every liveness claim is bounded the same way: baked at a commit.
